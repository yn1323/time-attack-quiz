# サンリオキャラクタークイズ仕様

## 概要

`docs/sanrio-characters.json` のデータを元に、200問のサンリオキャラクタークイズを作成する。

## 出力ファイル

`public/data/quizzes/サンリオキャラクター.json`

## クイズ形式

3択クイズ（既存の雑学.jsonと同じ形式）

```json
{
  "question": "問題文",
  "choices": ["選択肢1", "選択肢2", "選択肢3"],
  "answer": 0,
  "relatedLinks": ["https://www.sanrio.co.jp/characters/xxx/"]
}
```

---

## 問題パターン

### パターンA: キャラ → 属性を答える

| カテゴリ | 例 |
|---------|-----|
| 誕生日 | 「ハローキティの誕生日は？」→ 11月1日 / 3月6日 / 1月18日 |
| 好きな食べ物 | 「シナモロールの好物は？」→ シナモンロール / バナナアイス / らっきょう |
| 趣味 | 「ポムポムプリンの趣味は？」→ くつ集め / 日記をつける / お昼寝 |
| 種族 | 「ポチャッコは何の動物？」→ イヌ / ネコ / クマ |
| 出身地 | 「ハローキティの出身地は？」→ イギリス / フランス / 中国 |
| 家族 | 「キティの双子の妹の名前は？」→ ミミィ / ララ / サクラ |
| 友達 | 「シナモロールの親友は？」→ カプチーノ / チョッピ / バク |

### パターンB: 属性 → キャラを答える

| カテゴリ | 例 |
|---------|-----|
| 誕生日 | 「誕生日が2月29日のキャラは？」→ ポチャッコ / キティ / シナモン |
| 好きな食べ物 | 「バナナアイスが好物のキャラは？」→ ポチャッコ / プリン / けろっぴ |
| 趣味 | 「タップダンスが得意なキャラは？」→ ペックル / キティ / クロミ |
| 種族 | 「半魚人のキャラは？」→ ハンギョドン / タキシードサム / けろっぴ |
| 出身地 | 「南極出身のキャラは？」→ タキシードサム / シナモン / キキ |
| 特徴 | 「自称マイメロディのライバルは？」→ クロミ / ピアノちゃん / ミミィ |

## 作成ルール

1. **3択の選択肢**: 正解1つ + 不正解2つ（他キャラの属性を使用）
2. **relatedLinks**: 各問題に該当キャラのサンリオ公式URLを付与
3. **難易度バランス**: 有名キャラ（キティ、シナモン等）と知名度低めのキャラを混ぜる
4. **nullデータ**: 属性がnullのキャラはそのカテゴリでは出題しない

---

## 難易度UP: 紛らわしい選択肢

選択肢を「似たもの」で揃えて、正解を見つけにくくする。

| カテゴリ | 紛らわしくする方法 | 例 |
|---------|-------------------|-----|
| 誕生日 | 同じ月 or 近い日付 | 「11月1日 / 11月15日 / 10月31日」 |
| 好きな食べ物 | 同じジャンル（スイーツ同士など） | 「アップルパイ / プリン / ホットケーキ」 |
| 種族 | 似た動物 | 「イヌ / ネコ / クマ」 |
| 出身地 | 同じ地域 or ファンタジー系 | 「イギリス / フランス / ドイツ」 |
| キャラ名 | 似た名前 or 同じ作品 | 「キキ / ララ / ミミィ」 |
| 友達・家族 | 同じキャラの関係者 | 「カプチーノ / モカ / エスプレッソ」（全員シナモンの友達）|

---

## 選択肢のランダム表示

クイズデータでは `answer` を正解インデックスとして保持するが、**アプリ側で表示時にシャッフル**する。

relatedLinksのルール
問題文に登場するキャラ + 選択肢すべてのキャラの公式URLを含める
例: 「シナモロールの好物は？」選択肢「シナモンロール / バナナアイス / プリン」 → シナモロールのURL（問題文）のみ（選択肢は食べ物なのでURLなし）
例: 「バナナアイスが好物のキャラは？」選択肢「ポチャッコ / シナモロール / プリン」 → ポチャッコ、シナモロール、プリンのURL（選択肢すべて）
